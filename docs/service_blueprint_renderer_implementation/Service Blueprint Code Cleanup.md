# Service Blueprint Code Cleanup

## Identified Issues

1. `[High]` The staged pipeline is leaking `service_blueprint` semantics through CSS/view-specific backdoors instead of typed scene contracts. [serviceBlueprint.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprint.ts#L171), [serviceBlueprint.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprint.ts#L310), [macroLayout.ts](/home/knut/projects/sdd/src/renderer/staged/macroLayout.ts#L1382), and [serviceBlueprintRouting.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprintRouting.ts#L315) all reconstruct meaning from `classes` or `viewId`. That is a brittle architecture pattern: styling tokens have become semantic API, so innocuous class-name changes can break layout and routing.

2. `[High]` Lane titles and separators are bolted on after routing instead of being part of the scene contract. [serviceBlueprintRouting.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprintRouting.ts#L3661) clears `decorations`, and every public render entrypoint has to remember to re-run [attachServiceBlueprintDecorations in serviceBlueprint.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprint.ts#L310) at [serviceBlueprint.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprint.ts#L482), [serviceBlueprint.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprint.ts#L521), [serviceBlueprint.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprint.ts#L564), and [serviceBlueprint.ts](/home/knut/projects/sdd/src/renderer/staged/serviceBlueprint.ts#L595). That makes intermediate scenes structurally incomplete and easy to misuse.
